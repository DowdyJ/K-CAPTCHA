About KB-CAPTCHA

As bot detection technology has evolved, so too have the abilities of bots and their makers to bypass these systems. While not all bots are harmful, it is important that service providers and webhosts can choose when and how to restrict bot traffic.

To combat bots, website owners frequently turn to CAPTCHA as a solution. Often times, this is a varation of the image CAPTCHA. Users are to look at pictures and try to distinguish between images. However, this introduces several problems.

  • These image CAPTCHAs are unpleasant to solve.
  • Many services exist to automate the solving of CAPTCHAs both with AI and real humans.

As a result, real users are forced to solve these troublesome puzzles while bots pass freely. The KB-CAPTCHA project offers an alternate implementation. By analyzing users' actions while using a service, keyboard input in particular, we can make a guess as to whether a user is a bot or not without the user being forced to do a puzzle. This then can be used to serve a traditional image based CAPTCHA to users who appear to be bots without burdening other users.

Features

  • Uses machine learning to simultaneously analyze several different dimmensions of user keyboard input
  • Functionality exposed through REST API
  • Completely Dockerized (cross-platform)
  • GPL-Licensed (free!)
